# 배포 가이드 (Vercel)

지인들이 링크로 접속해 쓸 수 있게 인터넷에 올리는 방법입니다.
**핵심: 배포 환경에서는 구독 토큰이 아니라 `ANTHROPIC_API_KEY`(종량 과금)로 동작합니다.**
(코드는 이미 두 경로를 지원 — 로컬은 구독 토큰, 배포는 API 키를 자동 선택)

---

## 0. 준비물
- Node.js (설치돼 있음)
- **Anthropic API 키**: https://console.anthropic.com → **API Keys** → *Create Key* → `sk-ant-...` 복사
  - 결제수단/크레딧을 등록해야 실제 호출됩니다. (콘솔에서 월 사용 한도도 설정 가능 → 비용 폭탄 방지)

---

## 1. Vercel CLI 설치 & 로그인
```bash
npm i -g vercel
vercel login        # 브라우저로 이메일/깃허브 인증
```

## 2. 첫 배포 (프리뷰)
```bash
cd C:\ClaudeCodeProjects\saju
vercel
```
질문이 나오면 대부분 기본값(Enter)으로:
- *Set up and deploy?* → **Y**
- *Which scope?* → 본인 계정
- *Link to existing project?* → **N**
- *Project name / directory* → 기본값
- *Modify settings?* → **N** (Next.js 자동 감지)

→ 끝나면 **프리뷰 URL**이 나옵니다.

## 3. API 키 등록 (환경변수)
```bash
vercel env add ANTHROPIC_API_KEY
```
- 값: 복사한 `sk-ant-...` 붙여넣기
- 적용 환경: **Production**(원하면 Preview도) 선택

> 대시보드로도 가능: Vercel → 프로젝트 → **Settings → Environment Variables**

## 4. 프로덕션 배포
```bash
vercel --prod
```
→ **프로덕션 URL**(예: `https://saju-xxxx.vercel.app`)이 나옵니다. **이 링크를 지인에게 공유**하세요.
> 환경변수를 추가/변경한 뒤에는 반드시 다시 `vercel --prod` 로 재배포해야 반영됩니다.

---

## 참고 / 주의
- **로컬 비밀은 안 올라감**: `.env.local`(구독 토큰)은 `.vercelignore`로 업로드에서 제외됩니다. 배포본은 오직 `ANTHROPIC_API_KEY`로만 호출합니다.
- **비용**: 풀이 1건당 대략 수백 원 이하(Sonnet + 추론끄기 + 수천 토큰 기준). 지인 테스트 규모면 소액입니다. Anthropic 콘솔에서 **사용량/월 한도**를 꼭 확인·설정하세요.
- **함수 제한**: 서버 함수 최대 실행 60초로 설정돼 있습니다. 스트리밍 풀이는 보통 이 안에 끝납니다. (Vercel Hobby 한도 기준)
- **커스텀 도메인**: Vercel 대시보드 → Domains 에서 연결 가능.

## (대안) GitHub 연동 배포
자동 재배포를 원하면:
1. 이 폴더를 GitHub 저장소로 push
2. Vercel 대시보드 → *Add New Project* → 저장소 선택 → Import
3. Environment Variables에 `ANTHROPIC_API_KEY` 추가 → Deploy
이후 `git push` 할 때마다 자동 배포됩니다.

## 문제 해결
- **빌드는 되는데 함수 크기 초과 에러**가 나면, 로컬 전용 Agent SDK가 서버 번들에 포함된 경우입니다. 그때 알려주시면 배포 번들에서 제외하도록 조정하겠습니다.
- **호출 시 401/인증 에러**: `ANTHROPIC_API_KEY`가 Production 환경에 등록됐는지, 재배포했는지 확인하세요.
